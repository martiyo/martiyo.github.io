Configurar TP-Link como router principal detras del modem

Configurar el TP-Link como router principal detrás del módem, no como puente, así mantener todo el control de la red.

# Arquitectura recomendada:

g

Internet → Módem Coaxial (192.168.10.1) → TP-Link OpenWRT (WAN + LAN) → Dispositivos (pcs, notebooks, telefonos)

  • El TP-Link hará:
    1. NAT doble
    2. DHCP para la red local
    3. Firewall y control parental (horarios, bloqueos)
    4. DNS filtering (bloqueo de sitios)

Configuración paso a paso:

  • Configura la red LAN del TP-Link (tu red interna):
    1. Cambiar la IP de LAN para que no conflictúe con el módem (192.168.10.x) 
    uci set network.lan.ipaddr='192.168.1.1'
uci set network.lan.netmask='255.255.255.0'
uci commit network
    1. Configurar el WAN (conexión al módem):

    Configurar WAN para recibir IP del módem por DHCP

    uci set network.wan.proto='dhcp'
uci commit network

    Reiniciar la red

    /etc/init.d/network restart
    1. Conexión física:

    Puerto WAN del TP-Link → cualquier puerto del módem coaxial Puertos LAN del TP-Link → tus dispositivos (PC, telefonos via WiFi, etc)

    1. Verificar que funcione:

    Desde el router: ping 8.8.8.8 # Debe funcionar
